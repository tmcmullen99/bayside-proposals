Republish bids to pull in new swatches

Already-published bids are immutable HTML snapshots — new material swatches in the catalog don't reach them automatically. Select bids below and republish to bake the new swatches into a fresh snapshot. Old snapshots stay in place; each republish creates a new version slug.

How it works: Each republish runs the same flow as clicking "Publish new version" in the editor — pulls fresh data from the catalog, generates HTML, writes a new published_proposals row. Bids are processed one at a time. Plan ~5–10 seconds per bid; don't close this tab while it's running.
